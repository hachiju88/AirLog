# UI/UX 仕様書 (UI/UX Specification)

## 1. デザインコンセプト (Design Concepts)

### 「3タップ以内の記録」 (Speed)
アプリを開いてから写真撮影・音声入力完了までを最速にするUI設計。入力のハードルを極限まで下げる。

### 「親しみやすさ」 (Affinity)
AIからのフィードバックを画面中央などの目立つ位置に配置。無機質な数字管理ではなく、パートナーと共に歩む体験を作る。

### 「データ密度」 (Information Density)
スマートフォン縦持ち（Portrait）状態で、主要なトレンドや相関関係が把握できるグラフ設計。横持ちを強制しない。

## 2. 画面一覧と主要コンポーネント

### ① ダッシュボード (Home)
アプリのメイン画面。現在のステータスとAIの「褒め」が真っ先に目に入る。

*   **AIフィードバック・バブル**:
    *   今日の達成度や前日比較を元に、Geminiが生成した温かいメッセージを表示。
    *   (例) 「目標カロリーまであと200kcal！お昼のサラダが効いてますね🥗」
*   **今日のサマリー (Progress Ring)**:
    *   カロリー摂取 vs 消費のバランスをリングまたはバーチャートで直感的に表示。
*   **クイックアクション (FAB / Bottom Sheet)**:
    *   ワンタップで入力モードへ移行可能なボタン群。

### ② ログ入力・プレビュー画面 (Input & Review)
写真や音声の解析結果を確認・修正する画面。

*   **解析中のLoading State**:
    *   「思考中...」のアニメーションやスケルトン表示を実装し、待機ストレスを軽減。
*   **解析結果カード**:
    *   AIが特定した食品名、カロリー、PFCをカード形式で提示。
*   **クイック編集**:
    *   数値（g, kcal, 回数）をタップしてインライン編集可能にする。
*   **手動追加**:
    *   解析漏れ補助用のテキスト入力フォーム。

### ③ トレンド分析画面 (Analytics)
実績を振り返るための可視化画面。

*   **期間切り替え**:
    *   [日] [週] [月] [半年] [1年] [5年]
    *   横スクロール可能なチップ、またはドロップダウンメニューを採用。
*   **インタラクティブ・チャート**:
    *   Rechartsを使用。
    *   **体重推移**: 折れ線グラフ + BMIゾーン背景。
    *   **PFCバランス**: 積層棒グラフ。
*   **インサイト**:
    *   「先週同曜日比 -0.5kg」などの比較テキストを表示。

### ④ デバイス連携・設定画面 (Settings)
*   **Bluetoothステータス**: 接続状況と「計測開始」ボタン。
*   **プロフィール**: 身長、目標、AIパーソナリティ設定。

## 3. ナビゲーション設計 (Navigation)
アプリ下部に固定の **Bottom Navigation Bar** を配置し、主要画面間のスムーズな移動を実現する。

*   **[🏠 Home]**: ダッシュボードへ移動
*   **[📊 Analytics]**: 分析画面へ移動
*   **[⚙️ Settings]**: 設定画面へ移動
*   **(Central Action)**: 中央に強調された「＋」ボタンを配置し、記録メニュー（📸文字/🎤音声/✏️手動）を展開。

## 4. UIコンポーネント選定
*   **Library**: `shadcn/ui` (Radix UI + Tailwind CSS)
*   **Theme**:
    *   **System Gradient**: `from-indigo-100 via-purple-100 to-pink-100` をヘッダー、ナビゲーション、カード背景に統一適用し、没入感のあるブランド体験を提供。
    *   **Input**: 白背景の入力フィールドを採用し、グラデーション上での可読性を確保。
*   **Typography**: 可読性の高いサンセリフ体 (Inter, Noto Sans JP)。
